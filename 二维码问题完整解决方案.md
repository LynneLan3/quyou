## 🎯 技术原理

### 为什么数据会损坏？

1. **图片压缩**：
   - 分享卡片生成时，PNG 图片可能被压缩
   - 二维码的小方块可能模糊或失真

2. **扫描误差**：
   - 手机扫描时的角度、光线、距离影响识别
   - 低质量的二维码容易被误读

3. **纠错能力不足**：
   - M 级纠错只能恢复 15% 的损坏数据
   - 当损坏超过 15% 时，解码失败

### H 级纠错的优势

**纠错级别对比**：
- L (7%)：理想环境，无遮挡
- M (15%)：一般使用，轻微遮挡
- Q (25%)：可能有遮挡或损坏
- **H (30%)**：严重遮挡或需要嵌入 Logo ✅

**H 级的代价**：
- 二维码更密集（需要更多空间存储纠错数据）
- 但我们的 URL 不长，完全可以承受

### 为什么增大尺寸？

**尺寸与识别率的关系**：
```
小尺寸 (80px)  → 方块小 → 容易模糊 → 识别率低
大尺寸 (104px) → 方块大 → 清晰可辨 → 识别率高
```

**最佳实践**：
- 手机扫描：建议至少 100x100px
- 打印扫描：建议至少 200x200px

### 为什么提高像素比？

**像素比的作用**：
```
pixelRatio=1: 96px → 96px 图片
pixelRatio=2: 96px → 192px 图片
pixelRatio=3: 96px → 288px 图片 ✅
```

**优势**：
- 高分辨率屏幕显示更清晰
- 缩放时不失真
- 扫描识别率更高

## 🎓 经验总结

### 关键教训

1. **二维码质量至关重要**：
   - 不要为了美观牺牲功能
   - 尺寸、纠错级别、图片质量都很重要

2. **特殊字符要小心**：
   - URL 中的点号可能被误处理
   - 使用 URL 安全字符（下划线、连字符）

3. **测试要全面**：
   - 不同的扫描工具
   - 不同的环境条件
   - 不同的设备

4. **纠错级别的选择**：
   - 不要吝啬纠错级别
   - H 级虽然密集，但可靠性高
   - 对于重要功能，宁可牺牲一点效率

### 最佳实践

1. **二维码生成**：
   - 尺寸：至少 100x100px
   - 纠错：H 级（30%）
   - 内边距：启用
   - 对比度：黑白高对比

2. **图片质量**：
   - 像素比：3x
   - 质量：1.0（最高）
   - 格式：PNG（无损）

3. **URL 设计**：
   - 尽量简短
   - 使用 URL 安全字符
   - 避免特殊符号
