## 🎯 项目目标

创建一个基于兴趣问卷的社交匹配平台，帮助用户：
1. 通过问卷发现自己的兴趣类型
2. 找到志同道合的朋友
3. 通过分享功能邀请好友参与
4. 查看匹配度和详细的答案对比

## 🐛 主要问题与解决

### 问题 1: 分享链接无法打开
**症状**：点击分享链接后页面空白，报错 `Expected a JavaScript module but got text/html`

**原因**：
1. Vercel 的 `rewrites` 配置将所有请求重定向到 `index.html`
2. Vite 的 `base: './'` 导致使用相对路径

**解决**：
1. 将 `rewrites` 改为 `routes`，添加 `filesystem` 处理器
2. 将 `base` 改为 `'/'`，使用绝对路径

### 问题 2: 二维码扫描后 URL 格式错误
**症状**：扫描后 `from` 参数显示为 `iYTFi...` 而不是 `f.YTFi...`

**原因**：点号 (`.`) 在二维码编码/解码时被误处理

**解决**：将前缀从 `f.` 改为 `f_`（下划线更安全）

### 问题 3: 二维码扫描后数据损坏
**症状**：扫描后 URL 中的数据不完整或错误

**原因**：
1. 二维码尺寸太小（80px）
2. 纠错级别不足（M 级，15%）
3. 图片质量不够高

**解决**：
1. 增大二维码尺寸到 104px
2. 提升纠错级别到 H (30%)
3. 提高图片质量（pixelRatio=3, quality=1.0）

## 📊 技术亮点

### 1. 分享链接加密
```typescript
// 编码：UUID → Base64URL + 前缀
export function encodeFromParam(userId: string): string {
  const base64 = btoa(unescape(encodeURIComponent(userId)));
  const safe = base64.replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '');
  return 'f_' + safe;
}

// 解码：Base64URL → UUID
export function decodeFromParam(encoded: string | null): string | null {
  if (!encoded) return null;
  let raw = encoded.startsWith('f_') ? encoded.slice(2) : encoded;
  const base64 = raw.replace(/-/g, '+').replace(/_/g, '/');
  const padded = base64 + '='.repeat((4 - (base64.length % 4)) % 4);
  return decodeURIComponent(escape(atob(padded)));
}
```

## 🎓 经验总结

### 技术经验

1. **SPA 部署配置很重要**
   - 错误的路由配置会导致静态资源加载失败
   - 必须区分静态文件和页面路由

2. **二维码质量至关重要**
   - 尺寸、纠错级别、图片质量都会影响识别率
   - 不要为了美观牺牲功能性

3. **URL 安全字符的选择**
   - 避免使用特殊字符（如点号）
   - 优先使用下划线、连字符等安全字符

4. **调试日志的价值**
   - 详细的日志可以快速定位问题
   - 在关键流程添加日志很有必要

5. **文档化很重要**
   - 完整的文档可以帮助快速排查问题
   - 记录问题和解决方案对未来很有价值

### 开发流程

1. **问题分析**
   - 收集完整的错误信息
   - 分析问题的根本原因
   - 不要只解决表面问题

2. **解决方案**
   - 考虑多种方案
   - 选择最可靠的方案
   - 考虑向后兼容性

3. **测试验证**
   - 全面测试修复效果
   - 考虑边界情况
   - 多种环境测试

4. **文档记录**
   - 记录问题和解决方案
   - 更新 CHANGELOG
   - 提供验证步骤
